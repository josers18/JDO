# AGENTS.md - Customer_Documents

Context for AI coding agents working on the **Cumulus Bank customer-document catalog**.

This is a demo document-generation project, not a Salesforce DX package. There is no `sfdx-project.json`, no `force-app/`, and no deploy step. The project exists to generate customer-related PDFs that can support JDO demos, banker workflows, and generated-document experiments.

## Project rules

- Keep the generation architecture close to `Cumulus_Products`: shared ReportLab module, generator-owned outputs, local docs, artifact inventory, and demo disclaimers.
- Do not copy the Cumulus_Products brochure style. These are content-heavy operating documents, so favor dense narrative, evidence tables, controls, and appendices.
- Put reusable layout and document primitives in `generator/brand.py`.
- Put document content and document specs in `generator/generate_documents.py`.
- Generated PDFs live under `documents/<category>/`.
- `documents/README.md` and `docs/ARTIFACTS.md` are generated by `generator/generate_documents.py`.
- Salesforce-backed KYC PDFs are generated by `generator/generate_kyc_documents.py`.
- KYC PDF file names must follow `<AccountId>_KYC_<YYYY-MM-DD>.pdf`.
- Salesforce-backed Articles of Incorporation PDFs are generated by `generator/generate_articles_of_incorporation.py`.
- Articles PDFs must only use non-person Account records and file names must follow `<AccountId>_Articles_of_Incorporation_<YYYY-MM-DD>.pdf`.
- Articles PDFs intentionally do not use `generator/brand.py` or the shared Cumulus document template.
- Do not include real customer data, account numbers, tax IDs, credentials, or production communication language.

## Commands

```bash
cd Customer_Documents
python3 generator/generate_documents.py
python3 generator/generate_kyc_documents.py --target-org jdo-uqj0jr --limit 10
python3 generator/generate_articles_of_incorporation.py --target-org jdo-uqj0jr --limit 10
python3 -m unittest discover -s tests
```

If `reportlab` is missing:

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install reportlab
python3 generator/generate_documents.py
```

## Current catalog

- `01_Onboarding`: Retail Welcome Packet, Wealth Discovery Summary, Commercial Onboarding Checklist
- `02_Relationship_Review`: Retail Financial Snapshot, Wealth Annual Review, Commercial Relationship Review
- `03_Service_and_Retention`: Retail Service Follow-Up, Wealth Planning Next Steps, Commercial Treasury Readiness Brief
- `04_KYC`: Salesforce-backed KYC profiles generated per selected Account
- `05_Articles_of_Incorporation`: Salesforce-backed legal-form documents generated per selected non-person Account

## Common mistakes

- Editing generated PDFs manually. Update the generator and rerun it.
- Adding new document primitives inside `generate_documents.py`. Add reusable flowables to `brand.py`.
- Weakening the fictitious-bank and generated-demo-data disclaimers.
- Treating generated scorecards or next best actions as approved recommendations.
- Adding sensitive data to examples or test fixtures.
- Running full-org KYC generation casually. The current default org has 36K+ Accounts; use `--limit`, `--where`, or `--account-id` for smoke tests and `--all` only when the large output set is intended.
- Running full Articles generation casually. The current default org has 10K+ non-person Accounts; use a smoke test first and `--all` only when that output set is intended.
