# AGENTS.md — Snowflake_BoardEx_ExecIntel

Synthetic BoardEx / Equilar / ISS-style board director and executive intelligence dataset for the Cumulus FSC demo. One of 13. **Account-scoped with monthly cadence — the smallest Cumulus audience by 4.1× (Commercial Banking only, ~960 anchors), dethroning Plan 8 (MGP Financial Plans, 3,920) from that title. The first dataset where the SAMPLE_ANCHORS L1 fixture has *zero* relevant cohort members, forcing an inline synthetic-fixture override in the L1 conftest.**

## Boundaries
- Owns: `FINS.PUBLIC.BOARDEX_EXEC_INTEL`, `SP_GENERATE_BOARDEX_EXEC_INTEL`, `TASK_MONTHLY_BOARDEX_EXEC_INTEL`, and the DC Data Stream / DLO / DMO that federates this table.
- Does NOT own: `V_ACCOUNT_ANCHORS`, `MASTER_ACCOUNTS`, the seed/coverage helpers — see `Snowflake_Cumulus_Common`.
- Does NOT own any outbound Snowflake share. DC reads through via the existing "Snowflake (Federate / Zero Copy)" connector.
- **Account-scoped** — rows are keyed by composite PK `(ACCOUNT_ID, PROFILE_MONTH)`, with FK `ACCOUNT_ID` to `ssot__Account__dlm`. **1:1 — one row per distinct Commercial Banking account per month.**

## Conventions
- The SP uses `cumulus_common.seed_for(...)` for determinism with salt `"boardex"` (**month-bucketed**, single-salt — no year-stable streams; matches Plan 8's simpler shape).
- **One seed stream per anchor per run** — `seed_for(account_id, "boardex", month_start)` where `month_start = run_ts.replace(day=1, hour=0, minute=0, second=0, microsecond=0)`. Drives BOARD_SIZE, BOARD_INDEPENDENCE_PCT, WOMEN_BOARD_PCT, MINORITY_BOARD_PCT, BOARD_AVG_TENURE_YEARS, CEO_TENURE_YEARS, EXEC_TURNOVER_FLAG, GOVERNANCE_RATING, INTERLOCK_COUNT, KEY_DIRECTOR_NAME, RECENT_GOVERNANCE_EVENT_DATE, LAST_DATA_REFRESH_DATE. Mid-month re-runs are byte-identical because `month_start` truncates day + time.
- The SP uses `cumulus_common.assert_coverage(session, expected_sql, actual_sql)` for the standard 1:1 audience-vs-actual coverage check — distinct accts = audience (every distinct Commercial Banking anchor must produce exactly one row per month).
- The MERGE replaces on composite PK `(ACCOUNT_ID, PROFILE_MONTH)`. Re-runs same calendar month are idempotent (byte-identical). Re-runs in a later month INSERT new rows for that month.
- Audience SQL is `SELECT DISTINCT * FROM V_ACCOUNT_ANCHORS WHERE CLIENT_CATEGORY = 'Commercial Banking'`. 960 distinct accounts (probed 2026-05-28). Both `EMPLOYEE_COUNT` and `ANNUAL_REVENUE` expected ~100% populated for this BUSINESS-skewed cohort; `INDUSTRY` populated rate should be re-checked at audience-probe time and logged in scaffold notes. The SP probes `WHERE TRIM(CLIENT_CATEGORY) = 'Commercial Banking'` defensively in case casing/whitespace drift surfaces (one of the v1.5 9-value CLIENT_CATEGORY caveats).
- `accounts_processed` in `TASK_EXECUTION_LOG` equals `len(audience)` — the **distinct customer count** AND the row count (1:1 emit per anchor per month).

## Tests
- L1 (pytest): determinism in the same calendar month (byte-identical at 03:00 vs 23:30, day-1 vs day-15), input-shape validation (ACCOUNT_ID non-empty), boring case (mid-market enterprise: ~2,500 employees, INTERLOCK_DEGREE=2 → BOARD_SIZE 8-10, GOVERNANCE_RATING `Strong` or `Adequate`, EXEC_TURNOVER_FLAG mostly False, RECENT_GOVERNANCE_EVENT_DATE NULL ~70% of the time), **per-anchor range invariants** (every row's BOARD_SIZE in [5,15]; BOARD_INDEPENDENCE_PCT, WOMEN_BOARD_PCT, MINORITY_BOARD_PCT all in [0.0,100.0]; BOARD_AVG_TENURE_YEARS in [1.0,20.0]; CEO_TENURE_YEARS in [0.0,25.0]; INTERLOCK_COUNT in [0,5]), **per-row vocabulary invariants** (GOVERNANCE_RATING in `{Excellent, Strong, Adequate, Weak, Concerning}`; EXEC_TURNOVER_FLAG is a Python `bool`), **date-coherence invariants** (LAST_DATA_REFRESH_DATE ≤ run_ts.date(); when populated, RECENT_GOVERNANCE_EVENT_DATE ≤ run_ts.date()), **bias-band invariants** (EMPLOYEE_COUNT≥10000 → BOARD_SIZE≥9 always; EMPLOYEE_COUNT<100 → BOARD_SIZE≤8 always; INTERLOCK_DEGREE≥4 → INTERLOCK_COUNT≥2 always), schema contract (14 keys per dict; KEY_DIRECTOR_NAME is non-empty `"<First> <Last>"`).
- **L1 conftest synthetic-fixture-override pattern.** Plan 10 is the first Cumulus dataset where `SAMPLE_ANCHORS` has *zero* relevant cohort members — Commercial Banking is ~2.6% of the anchor pool and the 100-anchor Retail/Wealth/Household/Small Business slice contains none. The L1 conftest **does not consult `SAMPLE_ANCHORS` for cohort tests**. Instead it defines an inline `COMMERCIAL_BANKING_FIXTURE` of 5 hand-picked synthetic anchors covering the EMPLOYEE_COUNT / INTERLOCK_DEGREE bias bands (mid-market enterprise / regulated bank / family business / recent IPO / large cap) and uses that as `in_audience_anchors` directly. Rolled across 6+ months, the 5-anchor fixture yields ~30 rows — sufficient to exercise all 5 governance-rating tiers and every bias band. The Plan 5/6 graceful-skip pattern is explicitly **not** acceptable here: skipping every cohort-specific test silently would defeat the point. Document this deviation in the SP module docstring.
- L2 (`tests/integration/`): deploys SP into a fixture-backed schema with `V_ACCOUNT_ANCHORS_FIXTURE` (5 Commercial Banking + 7 non-Commercial-Banking — audience filter excludes the 7); asserts coverage (`COUNT(DISTINCT ACCOUNT_ID) = 5`), idempotency on same-month re-run (ROWS_INSERTED=0), all 14 columns populated correctly, range and vocabulary invariants enforced as in L1, 0 future-dated LAST_DATA_REFRESH_DATE, 0 future-dated RECENT_GOVERNANCE_EVENT_DATE when populated, at least 3 of the 5 GOVERNANCE_RATING tiers present (sampled across 5 anchors × 3 months).
- L3 (manual smoke, post-deploy): one CALL against jdo-uqj0jr; row count ≈ 960 (distinct Commercial Banking accounts); 5-row sample plausibility (mix of governance ratings); GOVERNANCE_RATING distribution skewed toward Strong / Adequate; range invariants (BOARD_SIZE in [5,15]; independence/diversity pcts in [0,100]; etc.); vocabulary all in canonical 5-value set; 0 future-dated LAST_DATA_REFRESH_DATE; 0 future-dated RECENT_GOVERNANCE_EVENT_DATE when populated; RECENT_GOVERNANCE_EVENT_DATE populated rate ≈ 30% (target 30%, tolerance ±5%).

## Gotchas
- **Smallest Cumulus audience by 4.1×.** 960 anchors vs Plan 8's 3,920 (next-smallest). SP runtime expected <1s. Storage trivial. **Per-anchor invariants are even more load-bearing than Plan 8** because the cohort is too narrow for any rate-convergence approach.
- **Cohort-fixture override (the headline structural deviation).** SAMPLE_ANCHORS' 100-anchor slice has zero Commercial Banking anchors. The L1 conftest **must** build an inline 5-anchor synthetic fixture (mid-market enterprise / regulated bank / family business / recent IPO / large cap, hand-picked to span EMPLOYEE_COUNT bias bands and INTERLOCK_DEGREE). Do not filter SAMPLE_ANCHORS, do not `pytest.skip` cohort tests à la Plan 5/6 — Plan 10 is the first dataset where graceful-skip would silently elide every cohort assertion. Document the deviation in the SP module docstring.
- **Per-anchor deterministic invariants, not distributional.** Property tests for range / vocabulary / bias-band / date-coherence are per-anchor or per-row (e.g. "every EMPLOYEE_COUNT≥10000 anchor has BOARD_SIZE≥9"), NOT mean comparisons or rate convergence. Plans 1-7 leaned on rate-convergence over SAMPLE_ANCHORS; Plan 8 first shifted to per-anchor invariants because the cohort was too narrow; Plan 10 takes that shift further because the cohort isn't even sampled in SAMPLE_ANCHORS.
- **Single NULLable column with independent NULL draw.** `RECENT_GOVERNANCE_EVENT_DATE` is NULL ~70% of rows via a flat 30%/70% Bernoulli draw — *no* multi-column coordination needed. Simpler than Plan 8's PLAN_STATUS-gated 2-NULL setup. The DDL marks it as NULL; the MERGE source SELECT must preserve `None` → `NULL`.
- **Anchor reads.** `EMPLOYEE_COUNT` (drives BOARD_SIZE bands and CEO_TENURE_YEARS bands) and `INTERLOCK_DEGREE` (drives INTERLOCK_COUNT). Both expected ~100% populated for Commercial Banking; the row factory falls back to `0` if NULL.
- **Date-coherence invariants** (must hold per row, asserted in L1 and L3):
  - `LAST_DATA_REFRESH_DATE ≤ run_ts.date()` — always, drawn 1-30 days ago.
  - `RECENT_GOVERNANCE_EVENT_DATE` (when populated) `≤ run_ts.date()` — drawn 1-365 days ago, no future-dated events.
- Salt is `"boardex"` (monthly cadence, single-salt). No year-stable subfield (board structure / governance rating / interlock count / dates are all month-bucketed).
- 1 BOOLEAN column (`EXEC_TURNOVER_FLAG`) — declare as `Boolean` in DC DLO + DMO (Plan 5 / Plan 8 finding — defaulting to Text fails the Boolean parse). Snowflake stores BOOLEAN natively; the MERGE source SELECT casts via `EXEC_TURNOVER_FLAG::BOOLEAN` to be explicit (write_pandas could mis-type on an empty target).
- 1 NULLable date column (`RECENT_GOVERNANCE_EVENT_DATE`) — independent 30% Bernoulli, no enum gating.
- The `write_pandas(auto_create_table=True)` mis-types `datetime64[ns]` as `NUMBER(38,0)`. The MERGE source SELECT casts back via `TO_TIMESTAMP_NTZ(GENERATED_AT::NUMBER / 1000000000)` — see template Task 4 §_merge.
- 14 columns total: 13 NOT NULL + 1 NULLable (RECENT_GOVERNANCE_EVENT_DATE). 1 BOOLEAN NOT NULL.
- The DLO → DMO field mapping must be completed in DC Setup UI for fully-custom DMOs (the API endpoint returns 500). See the recipe at `../Snowflake_Claritas_Demographics/docs/dc-setup-recipe.md` from Plan 1 T7.
- Snowflake DATE columns auto-discover as `MM/dd/yyyy` in the DC data-stream POST body. Use that format for `PROFILE_MONTH`, `RECENT_GOVERNANCE_EVENT_DATE`, `LAST_DATA_REFRESH_DATE`'s `sourceFields` entries, NOT `yyyy-MM-dd`.
- DC mapping descriptions must be ≤521 chars (Plan 6 finding). Trim the DMO description if drafting against the rowspec verbatim.
- "BoardEx" / "Exec Intel" are both clean — no `&` sanitize gymnastics needed (compare Plan 6's `D&B` workaround).
- `accounts_processed = len(audience) = row count` (1:1 emit per anchor per month) — same shape as Plans 1-3, 5, 7, 8.
- Cadence is **monthly** (`'USING CRON 0 7 1 * * UTC'` — first of the month at 07:00 UTC). Matches Plans 1-3, 6, 8.
- **Volume.** ~960 rows/month. Re-runs same month MERGE-replace in place; live storage grows by ~960 rows per month.
