### 2026-05-29 — DC_AgentForce_Markdown_Renderer + Cumulus_Assistant — Agentforce markdown rendering
- **`DC_AgentForce_Markdown_Renderer/`** (new DX project) — A reusable Lightning Web Component that renders Agentforce GenAI Function output as styled HTML, wired in via the platform's Lightning Type renderer-override mechanism (`c__markdownResponse` Lightning type → `c/markdownRenderer` LWC). No Flow plumbing or Apex sanitizer needed — the Agentforce panel binds renderer to type at runtime.
  - **Markdown parser** — regex-based handler for headings, **bold**, *italic*, ***triple-asterisk***, ordered + unordered lists, blockquotes, GitHub-flavored tables, fenced code blocks (with language tag) + inline code, and links. Code-fence content extracted to placeholders BEFORE escapeHtml so HTML inside fences renders as literal text.
  - **Security boundaries** — URL scheme allowlist (http(s):/mailto:/relative) with control-char stripping (defeats `java\tscript:` padding); curated tag emission; `escapeHtml` neutralization of `& < > "`; `rel="noopener noreferrer" target="_blank"` on every emitted `<a>`.
  - **Tests:** 38 Jest tests across 5 describe blocks (escapeHtml, URL allowlist, markdown constructs, value getter, edge cases). Includes security-critical URL allowlist tests covering `javascript:`, `data:`, `vbscript:`, `file:`, `ftp:` rejection plus padded variants (`java\tscript:`, `java\nscript:`, leading-whitespace, mixed-case). Run with `npm test`.
  - **Integration guide** at [`DC_AgentForce_Markdown_Renderer/docs/INTEGRATION_GUIDE.md`](DC_AgentForce_Markdown_Renderer/docs/INTEGRATION_GUIDE.md) covers four deployment surfaces (GenAI Function via output schema typing, custom LWC on record/app/home pages, programmatic embedding from another LWC, other Lightning Type renderer overrides), a markdown syntax support table, the security model, and troubleshooting.
- **`Cumulus_Assistant/`** (new DX project) — The Cumulus Bank Agentforce agent definition (3.3K-line `.agent` script with 18+ topic subagents covering Cases, communication, post-meeting follow-up, GeneralCRM, RecordManagement, ProductOffers, FinancialAccounts, ClientInsights, Referrals, etc.) and the `DC_Product_Offers` GenAiFunction (typed `c__markdownResponse`, consumes the renderer above). Lifted out of an earlier combined `markdown-renderer-project/` into its own DX root for clean reuse: the renderer is org-wide infrastructure, the agent is one of N consumers.
- **Rename + split provenance:** the original artifact landed as `markdown-renderer-project/` and was renamed `DC_AgentForce_Markdown_Renderer/` (matches the `DC_*_LWC` family naming). The agent bundle + GenAiFunction were lifted into a separate `Cumulus_Assistant/` project so the renderer surface is reusable across other agents.
- **Deploy ordering:** renderer project FIRST (creates the Lightning type), Cumulus_Assistant second (references the type). Live in `jdo-uqj0jr` as of 2026-05-29: deploy IDs `0Afam00002UtDvBCAV` (renderer) + `0Afam00002UtDwnCAF` (assistant).
- **HTML auto-detect feature attempted and reverted (2026-05-30/31).** A DOMParser-based HTML sanitizer was added in commit `8aa25573` and reflected in docs through commit `5a3c40cb`. The feature broke the Agent Authoring Builder save validator with a misleading "from c__markdownResponse to c__markdownResponse" error. Source LWC + docs reverted to the markdown-only state from commit `0914a056`. The Setup Agent Builder save error has not been root-caused; a Salesforce support case may be needed to recover the runtime type registration. See [`DC_AgentForce_Markdown_Renderer/docs/INTEGRATION_GUIDE.md`](DC_AgentForce_Markdown_Renderer/docs/INTEGRATION_GUIDE.md) Troubleshooting for the workaround path.
