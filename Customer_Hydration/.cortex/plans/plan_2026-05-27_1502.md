## Context

The `LOAD_MASTER_ACCOUNTS` failure was caused by duplicate rows in the source Data Cloud view (`ssot__Account__dlm`) — it has 37,445 rows but only 36,813 distinct `ssot__Id__c` values (632 duplicates). The deployed procedure in Snowflake was **already hot-fixed** with a `MERGE` + `ROW_NUMBER()` dedup pattern, and has been succeeding since. However, the **repo file is out of sync** — it still has the old naive `INSERT` version that doesn't handle duplicates.

The fix is to update the repo to match the deployed fix and add a structural safeguard (unique key on the table) so this class of error is permanently prevented.

## Implementation Steps

### 1. Update `procedures/sp_load_master_accounts.sql` — replace with the deployed MERGE version

Replace the simple DELETE+INSERT with the MERGE approach that:
- Uses `ROW_NUMBER() OVER (PARTITION BY "ssot__Id__c" ORDER BY "ssot__DataSourceId__c")` to deduplicate the source
- MERGEs into `MASTER_ACCOUNTS` on `ACCOUNT_ID` (update existing, insert new)
- Logs execution to `TASK_EXECUTION_LOG` with error handling via EXCEPTION block
- Returns a meaningful status message

### 2. Update `schemas/master_accounts.sql` — add a unique key constraint

Add a unique constraint on `(ACCOUNT_ID, SNAPSHOT_DATE)` as a structural guardrail. In Snowflake, unique keys are not enforced at DML time (they're informational), but they signal intent and enable optimizer behavior. Alternatively, since the current MERGE treats the table as a "latest state" table (single row per account, updating `SNAPSHOT_DATE` in place), change the DDL to reflect that `ACCOUNT_ID` is the natural key.

### 3. Add `schemas/task_execution_log.sql` — DDL for the logging table

The deployed Snowflake environment already has this table (the procedure writes to it). Add the DDL to the repo so it's tracked.

### 4. Update `AGENTS.md` — reflect current architecture

- Update the "Idempotency" section: SP_LOAD_MASTER_ACCOUNTS now uses MERGE (not snapshot-per-day INSERT)
- Note the TASK_EXECUTION_LOG table now exists
- Add a "Common mistakes" entry about Data Cloud source duplicates
